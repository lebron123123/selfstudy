# 讲义 02 · 整体架构与「渲染循环」大框架

> Day 2 用。这是**全产品最重要的一份讲义**。看懂这里，1900 行的 `index.html` 就从「一堵墙」变成「一排抽屉」。

---

## 一、核心思路：没有框架，怎么让页面动起来？

现代网站常用 React/Vue 等框架，但这个产品是**纯手写 JavaScript**，没有任何框架。它用一个极简、但非常清晰的套路来管理界面，叫「**状态驱动渲染**」：

```
  一堆全局变量（状态） ──▶ 函数把状态拼成一大段 HTML 字符串 ──▶ 塞进页面某个盒子 ──▶ 给新按钮绑上点击事件
        ▲                                                                                    │
        └──────────────────── 用户点按钮 → 改变量 → 再来一遍 ◀───────────────────────────────┘
```

一句话：**改变量 → 重画。** 界面永远是「当前状态」的一面镜子。你只要盯住两件事：**有哪些状态变量**、**重画时调用了谁**。

---

## 二、状态变量：整个 App 的「记忆」

看 `index.html` 第 535–552 行，所有全局状态都在这里（我加了中文说明）：

```javascript
const STEPS = ["选择领域","项目信息","财务测算","章节范围","逐章生成","复核与签发"]; // 可研生成的6步
let currentStep = 0;     // 可研生成走到第几步（0~5）
let domainKey = null;    // 选了哪个报告领域
let chapters = [];       // 当前报告的章节数组
let kbEntries = [];      // 用户上传的参考资料
let appMode = null;      // ★最顶层开关： null=首页 | 'report'=可研生成 | 'calc'=独立测算
let calcType = null;     // 测算类型： 'gaibao' | 'rent' | 'sale'
let scStep = 0;          // 测算模块走到第几步： 0选类型 1参数 2结果
let scResult = null;     // 测算结果（三个 *calc.js 算出来的东西）
let scParams = null;     // 测算输入参数
let aiChat = [];         // AI 问答历史
const project = { name:"", owner:"", industry:"", location:"", type:"", scale:"", desc:"" }; // 项目基础信息
```

**三个最关键的「开关」变量**，务必记牢：

- `appMode`：决定你在哪个大模块。`null` 首页、`'calc'` 财务测算、`'report'` 可研生成。
- `currentStep`：在可研生成里，控制 6 步流程走到哪。
- `scStep`：在财务测算里，控制 3 步（选类型→录参数→看结果）走到哪。

**理解全产品的钥匙**：任何一个界面，都是这几个变量某个组合的产物。比如「`appMode='calc'` 且 `scStep=1`」＝「财务测算的参数录入页」。

## 三、渲染循环：三个核心函数

### 1）`renderSheet()`（第 682–694 行）——总调度台

这是「重画」的入口。它根据状态变量，决定把哪段 HTML 塞进主区域 `#sheet`：

```javascript
function renderSheet(){
  const sheet = document.getElementById("sheet");           // 找到主内容盒子
  if(appMode===null){ sheet.innerHTML = stepHome(); ...; return; }        // 首页
  if(appMode==="calc"){ sheet.innerHTML = renderCalcModule(); ...; return; } // 财务测算
  if(currentStep===0) sheet.innerHTML = stepDomain();       // 以下是可研生成6步
  if(currentStep===1) sheet.innerHTML = stepProjectInfo();
  if(currentStep===2) sheet.innerHTML = stepCalc();
  if(currentStep===3) sheet.innerHTML = stepChapters();
  if(currentStep===4) sheet.innerHTML = stepGenerate();
  if(currentStep===5) sheet.innerHTML = stepReview();
  bindEvents();                                             // ★关键：给新画出来的按钮重新绑事件
  mountAnchorNav();
}
```

读法：
- `document.getElementById("sheet")`：在页面里找到 id 是 `sheet` 的那个 `<div>`（第 525 行）。这是「舞台」。
- `sheet.innerHTML = stepHome();`：`stepXxx()` 这些函数**返回一大段 HTML 字符串**（就是讲义 01 例 4 那种 `'...'+变量+'...'`），赋给 `.innerHTML` 就等于「把这段内容画到舞台上」。
- 每个 `stepXxx()` 就是「一个页面/一步」的画法。想看某一步长啥样，直接去读对应函数即可——**它们互相独立，可以一个个单独看**。

### 2）`renderTOC()`（第 660–678 行）——左边的步骤导航

同样是「读状态 → 拼 HTML → 塞进 `#tocList`」。它根据 `appMode` 决定显示「测算的 3 步」还是「可研的 6 步」，并高亮当前步（`i===scStep` 或 `i===currentStep`）。

### 3）`bindEvents()`（第 1638 行起）——把按钮接上电

**这是纯手写渲染最容易忽略、也最关键的一环。** 因为每次 `innerHTML = ...` 都会把旧 HTML 整个换掉，旧按钮上的点击事件也没了，所以每次重画后必须**重新给按钮绑定 `onclick`**。`bindEvents()` 就在干这件事，例如：

```javascript
const btn = document.getElementById("someBtn");
if(btn) btn.onclick = () => { currentStep++; renderTOC(); renderSheet(); };
```

看这一句就懂了整个循环：**点按钮 → 改状态（`currentStep++`）→ 调 `renderTOC()`+`renderSheet()` 重画**。回到第一节那张图。

> 财务测算模块的按钮由 `bindCalcEvents()`（第 937 行）单独绑，思想完全一样。

## 四、三条主路（全流程大框架）

从首页出发，`appMode` 把用户分到三条路：

```
                       首页 stepHome() (appMode=null)
                      /                              \
          点“财务测算”                            点“可研生成”
        appMode='calc'                          appMode='report'
              │                                        │
   renderCalcModule() 按 scStep 分3步            按 currentStep 分6步 (STEPS)
     0 选类型 scStepType()                       0 选领域   stepDomain()
     1 录参数 scStepForm()  ──调用──▶            1 填信息   stepProjectInfo()
     2 看结果 scStepResult()   nrcalc/rentcalc/  2 测算     stepCalc()
                                salecalc.js       3 选章节   stepChapters()
                                                  4 逐章生成 stepGenerate() ──▶ 调 /api/generate
                                                  5 复核签发 stepReview()   ──▶ 导出 Word
```

- **财务测算路**（`appMode='calc'`）：`renderCalcModule()`（第 751 行）根据 `scStep` 在 `scStepType/scStepForm/scStepResult` 之间切。算钱时调三个 `*calc.js`（讲义 03/04）。
- **可研生成路**（`appMode='report'`）：6 步依次是 `stepDomain → stepProjectInfo → stepCalc → stepChapters → stepGenerate → stepReview`（讲义 04）。

## 五、前端 ↔ 后端：什么时候「出门」找服务器

前端大部分事情自己在浏览器里就做了（画界面、算钱）。只有 3 类事必须请求后端（用 `await fetch("/api/...")`）：

| 前端函数 | 请求的后端接口 | 干什么 |
|---|---|---|
| `startApp` / `checkLogin`（第 1885/1901 行） | `/api/auth` | 登录 / 注册 |
| `cloudSaveNow` / `openProject`（第 1796/1873 行） | `/api/projects` | 云端存/取项目 |
| `askAI` / `generateSection`（第 915/1447 行） | `/api/generate` | 调 AI 写内容 |

这些 `/api/xxx` 就对应 `functions/api/xxx.js` 后端文件（讲义 05）。**记住这张表，Day 7 串链路时会用到。**

## 六、还有一个「防丢」小系统：草稿自动存档

第 555–603 行是一套「本地草稿」机制：每次改动都把状态 `JSON.stringify` 存进浏览器的 `localStorage`（`saveDraft`），刷新页面时 `loadDraft` 读回来问你「要不要恢复」。这是「断网/刷新不丢数据」的兜底。理解为「把状态变量打包存到浏览器硬盘」即可，不用深抠。
