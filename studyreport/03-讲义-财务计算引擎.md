# 上半 · 核心思路（Day 3）

## 一、这三个引擎是什么

它们是**纯计算模块**：输入一堆参数（面积、租金、贷款…），输出一整套逐年财务报表和指标（IRR、NPV…）。**不碰界面、不碰网络**，就是三个「计算器芯片」。前端在「结果页」调用它们，例如（`index.html` 第 891 行附近）：

```javascript
scResult = window.RentCalc.calc(scParams);   // 出租类
```

每个文件都用同一种「打包」方式暴露自己（`rentcalc.js` 第 2 行 / 第 199 行）：

```javascript
window.RentCalc = (function(){
  function calc(p){ ... }        // 核心函数
  return { calc };               // 只对外暴露 calc
})();
```

这叫 **IIFE（立即执行函数）+ 模块模式**：`(function(){ ... })()` 定义完立刻执行，内部函数是「私有」的，只把 `calc` 挂到全局 `window.RentCalc` 上。理解成 Python 里「一个模块只 `export` 一个 `calc` 函数」即可。

## 二、骨架七步（全产品财务逻辑的地图）

打开 `rentcalc.js`，只看那些 `===== N. xxx =====` 分节注释，你会看到 `calc(p)` 就是**顺序做七件事**：

| 步 | 行号 | 在算什么 | 核心思想 |
|---|---|---|---|
| 0 | 15–20 | **排年份** | 建设期 N 年 + 运营期 M 年，拼成 `allYears`，标记每年是不是运营年、几个月 |
| 1 | 22–37 | **收入** | 住宅租金 + 车位 + 其他，逐年算 |
| 2 | 39–64 | **经营成本** | 管理费、保险、维修、装修重置、折旧…共 8 项 |
| 3 | 66–94 | **还本付息** | 贷款逐年「期初→计息→还本→期末」滚动迭代 |
| 4 | 96–112 | **税金** | 增值税、附加、印花税、房产税、土地使用税等 |
| 5 | 114–121 | **总成本汇总** | 经营成本 + 运营期财务费用（不含税金、不含建设期财务费用） |
| 6 | 122–140 | **损益** | 利润总额 → 五年弥补亏损 → 所得税 → 净利润 |
| 7 | 142–181 | **现金流 + IRR/NPV** | 逐年净现金流、累计、折现、内部收益率、回本年 |

（注：源码里有 8 个 `===== N =====` 分节标记，我把「总成本汇总」和「损益」并读为一段思路，合起来正好是「七步骨架」。）

**Day 3 只要建立这张地图**：知道七步的先后和各自目的，不用看公式细节。

## 三、几个「财务 + 编程」都要懂的核心概念

- **逐年表**：所有量都是「按年份存的字典」，如 `income[2027]`、`cost[2027]`。可以想成一张 Excel，行是年份、列是各项。代码里用 `{}` 存，用 `allYears.forEach(y=>...)` 逐行填。
- **运营期 vs 建设期**：建设期在盖楼、没收入（`isOp[y]` 为 false）；运营期才有租金。很多公式对两者处理不同，所以到处见 `if(!isOp[y]) {...;return;}`。
- **年中折现**：算 NPV 时假设现金流发生在年中，所以折现指数是 `idx+0.5`（第 152 行）。这是财务惯例，不是 bug。
- **现金流出「不含折旧和财务费用」**：折旧是「账面摊销」不是真花钱，利息在别处单独处理，所以算「真实现金流出」时把它们排除（第 149 行注释）。这是财务测算的常见口径。
- **IRR（内部收益率）**：让「NPV=0」的那个折现率。没有公式直接解，只能**数值迭代逼近**（牛顿法），见下半。

# 下半 · 具体代码（Day 4）

现在逐节读 `rentcalc.js`。建议**左边开代码、右边开这份讲义**对照。

## 第 0 步 · 排年份（15–20 行）

```javascript
const buildYearsArr = Array.from({length:p.buildYears},(_,i)=>p.buildStart+i);  // 建设期年份列表
const opStart = p.buildStart + p.buildYears;                                     // 运营首年
const operateArr = Array.from({length:p.operateYears},(_,i)=>opStart+i);         // 运营期年份列表
const allYears = [...buildYearsArr, ...operateArr];                              // 全部年份
const isOp = {}; allYears.forEach(y=>isOp[y]=y>=opStart);                        // 每年是否运营年
const monthD = {}; allYears.forEach(y=>monthD[y]= isOp[y] ? (y===opStart?(p.firstMonths||12):12) : 12);
```

- `Array.from({length:n}, (_, i)=>...)`：生成长度 n 的数组，第 i 个元素由箭头函数算出。等价 Python `[f(i) for i in range(n)]`。`_` 表示「这个参数用不到」。
- `[...a, ...b]`：把两个数组展开合并（**扩展运算符**），等价 Python `a + b` 或 `[*a, *b]`。
- `isOp[y]=y>=opStart`：把「y 是否 ≥ 运营首年」这个 true/false 存进字典。
- `monthD`：运营首年可能只运营几个月（`firstMonths`），其余年 12 个月。这里用了嵌套三元表达式，拆开读：`是运营年吗? (是运营首年吗? firstMonths : 12) : 12`。

## 第 1 步 · 收入（22–37 行）

见讲义 01 例 3 已逐句拆过。核心两条公式：
- 住宅收入 = `面积 × 当年租金 × 当年出租率 × 月数 ÷ 10000`（÷10000 是「元 → 万元」）。
- 车位收入 = `车位数 × 车位月租 × 出租率 × 月数 × 收入系数 ÷ 10000`。
- 租金逐年递增：第 27 行 `p.rent * Math.pow(1+p.rentRate/100, Math.floor(idx/p.rentSpan))`——每过 `rentSpan` 年涨 `rentRate%`。`Math.floor(idx/span)` 是「涨了几次」。

## 第 2 步 · 经营成本（39–64 行）

八项成本逐年算。挑两个代表：
```javascript
const mgH = p.area*occ*12*1.92*p.manageCoeff/10000;   // 管理费(住房)
const dep = opIndex[y]<=50? p.totalInvestment*(1-0.2)/50 : 0;  // 折旧：总投资×80%÷50年
```
其中「装修重置费」（42–49 行）稍复杂：公租房每 20 年、保租房每 10 年要重置一次装修，费用在到期后 10 年里分摊。读懂这段的关键是那个 `for(let rn=resetPeriod; rn<=maxOpNum; rn+=resetPeriod)` 循环在「找每个重置周期」。第一遍可跳过细节，知道「它在把一笔大装修费摊到若干年」即可。

## 第 3 步 · 还本付息迭代（66–94 行）★重点

这是「迭代」思想的典型例子——**今年的期末余额，是明年的期初余额**：

```javascript
let endLast=0;                          // 上一年期末余额，初始0
allYears.forEach(y=>{
  const begin=endLast, cur=loanPlan[y]||0;        // 期初 = 上年期末；今年新借
  const interest=(begin+cur/2)*rate;              // 计息（新借按半年计息）
  const payInt=interest;                          // 利息当年全部付掉
  const rp = y<lastLoanYear? Math.min(repayPlan[y]||0, maxRepay) : maxRepay; // 还本
  let end=begin+cur+interest-payInt-rp; end=Math.max(end,0);   // 期末余额
  loan[y]={begin, borrow:cur, interest, repay:rp, payInt, end}; // 存这一年
  endLast=end;                                    // ★传给下一年
});
```

读法：把它想成一个「贷款账户」逐年滚动。`endLast` 这个变量就是「记忆」，每轮循环更新它、再喂给下一轮。Python 里写法一模一样。**这种「上一轮结果喂下一轮」的迭代，全项目多处出现（销售类的增值税累计抵扣也是），务必吃透。**

## 第 4 步 · 税金（96–112 行）

逐年算六个税种。注意两个「财务惯例」写法：
- **价税分离**：`resi*(0.015/1.05)`——含税收入里剥离出税额，`/1.05` 是去掉 5% 的价。
- **房产税前三年免征**：`if(opIndex[y]>3){ prop=... }`——只有运营第 4 年起才算房产税。

## 第 5 步 · 损益 + 五年弥补亏损（114–140 行）★重点

「弥补亏损」是税务规则：**盈利后可以先拿前 5 年的亏损来抵，抵完剩下的才交所得税**。代码用几个「记忆变量」实现：

```javascript
let lossHist=[], firstProfitYear=null, lastNeg=0, lossUsed=0;
allYears.forEach((y,idx)=>{
  const pt = r4(income[y].total - totalCost[y].total - tax[y].total);  // 当年利润总额
  lossHist.push(pt);
  if(firstProfitYear===null && pt>0) firstProfitYear=y;                // 记住首次盈利年
  let makeup=0;
  if(firstProfitYear!==null){
    if(lossUsed>=5) makeup=0;                                          // 最多抵5年
    else if(y===firstProfitYear) makeup=lossHist.slice(Math.max(0,idx-5),idx).reduce((s,v)=>s+v,0); // 首盈年：抵前5年亏损之和
    else makeup = lastNeg<0? lastNeg:0;                               // 之后：结转上年负的应纳税所得
  }
  const taxable=pt+makeup;                                            // 应纳税所得额
  const incomeTax = taxable>0? r4(taxable*0.25):0;                    // 所得税25%
  profit[y]={total:pt, makeup, taxable, incomeTax, net:r4(pt-incomeTax)};
});
```

三个「记忆变量」：`firstProfitYear`（首次盈利年）、`lossUsed`（已经抵了几年）、`lastNeg`（上年结转的负数）。逻辑绕，但**思路是「先攒亏损、盈利后逐年抵、最多抵五年」**。第一遍抓住这句话即可，第二遍再对着代码把三个变量的变化走一遍。

> `salecalc.js` 里这段几乎一字不差（出售类利润总额不减税金，是唯一差别），可对照着看，加深印象。

## 第 6 步 · 现金流 + IRR/NPV（142–181 行）★重点

```javascript
const cf={}; let cum=0, cumNpv=0;
allYears.forEach((y,idx)=>{
  const inflow=income[y].total;                                  // 流入=总收入
  const outflow=r4(invest + tax[y].total + c.mgH+c.mgP + ... + profit[y].incomeTax); // 流出（不含折旧/财务费用）
  const net=r4(inflow-outflow);                                  // 净现金流
  cum+=net;                                                      // 累计净现金流
  const npv=net/Math.pow(1+discount, idx+0.5);                   // 年中折现
  cumNpv+=npv;                                                   // 累计净现值
  cf[y]={inflow, outflow, net, cumNet:r4(cum), npv, cumNpv};
});
```

- **回本年**（第 167–168 行）：第一年「累计净现金流 ≥ 0」的年份，就是回本年。
- **IRR**（第 158 行调用 `excelIrr`，183–198 行实现）：牛顿迭代求「让 NPV=0 的折现率」。

## IRR 的牛顿迭代（183–198 行）——值得单独看

```javascript
function excelIrr(fl){
  if(!fl.some(f=>f>0)||!fl.some(f=>f<0)) return null;  // 必须有正有负才有解
  for(const g of [-0.01,-0.02,...,0.1]){                // 多个起点，防止不收敛
    let r=g;
    for(let i=0;i<maxIter;i++){
      const v=npvAt(r,fl);                               // 当前折现率下的NPV
      if(Math.abs(v)<tol) return r;                      // NPV≈0 → 找到IRR
      const d=(npvAt(r+h,fl)-v)/h;                        // 数值求导（斜率）
      let nr=r - v/d;                                     // 牛顿法：沿斜率往NPV=0挪
      r=nr;
    }
  }
  return null;
}
```

核心思想：**不断猜一个折现率 r，算它的 NPV，用斜率修正 r，直到 NPV 接近 0。** 试多个起点是因为亏损项目的曲线可能有多个解或难收敛。这段和 `salecalc.js`、`nrcalc.js` 里的 IRR 完全一致——**一份代码，三处复用**。

## 对照：三个引擎的异同（读完 rentcalc 再翻另两个）

| | nrcalc（非居改保） | rentcalc（出租） | salecalc（出售） |
|---|---|---|---|
| 收入主体 | 住宅租金 | 住宅+车位+其他 | 配保房**销售** + 商业出租**净现值** + 住宅/车位 |
| 特殊逻辑 | 收楼成本、装修摊销 | 装修重置(20/10年) | 地价抵减增值税、回收固定资产余值、调整所得税 |
| 骨架七步 | ✅ 同 | ✅ 同 | ✅ 同 |
| 损益/IRR | 同一套 | 同一套 | 同一套 |

**结论**：吃透 `rentcalc.js` 的七步，另两个只是「换了收入和几处税的公式」。这也是为什么当初做出售类时，是「照着骨架翻译公式」。

<details>
<summary>参考答案要点</summary>

1. 排年份→收入→成本→还本付息→税金→损益→现金流/IRR；折旧是账面摊销不是真实现金支出，故算真实现金流出时排除。
2. `[opStart+i for i in range(p.operateYears)]`。
3. 它保存「上一年期末借款余额」，作为下一年的期初；去掉就无法在年份间传递余额，迭代断裂。
4. 分别是：首次盈利的年份、已用于弥补的年数（上限5）、上一年结转的负应纳税所得额。
5. IRR 是「令 NPV=0」的折现率，NPV 是 r 的高次函数无解析解，只能数值逼近；多起点是为在亏损/非常规现金流下提高收敛成功率、避开无效解。
6. 出售类利润总额 = 总收入 − 总成本（**不再单独减税金及附加**），出租类要减；其余弥补亏损逻辑一致。
</details>
