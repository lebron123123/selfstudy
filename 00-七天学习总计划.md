# 七天吃透「可研报告与财务测算」产品 · 学习总计划

> 面向：会一点 Python、JS/前端不熟、每天 1–2 小时的你。
> 方法论（你定的，我完全采纳）：**先最基础语法 → 整体设计全流程大框架 → 每个点的核心思路 → 每个点的具体代码 → 整合复盘**。
> 这个「由浅入深」的顺序不只用在一周之间，也用在**每一份讲义内部**：每份讲义都先讲「核心思路」，再看「具体代码」。

## 先建立一张「地图」：这个产品由什么组成

打开项目文件夹，你会看到这些文件（共 19 个）。先不用懂，只要知道谁是谁。按「四大类」分：

**A. 前端页面（浏览器里跑的）**

| 文件 | 角色 | 类比 |
|---|---|---|
| `index.html` | 主前端：用户看到、点的一切（首页/测算/可研生成/登录/我的项目），约 1900 行 | 主程序 + 界面 |
| `admin.html` | **管理员后台**：可视化编辑各领域的章节大纲（写进云端，前台优先读它） | 后台管理页 |

**B. 三个财务计算引擎（纯计算，不管界面）**

| 文件 | 角色 | 类比 |
|---|---|---|
| `nrcalc.js` / `rentcalc.js` / `salecalc.js` | 非居改保 / 出租 / 出售 三类测算 | 三个独立「计算器芯片」 |

**C. 后端（跑在 Cloudflare 服务器上，`functions/api/` 目录）**

| 文件 | 角色 | 类比 |
|---|---|---|
| `_auth.js` | 共享安全工具：密码哈希、令牌签名/验签（文件名带`_`不对外） | 保安室的工具箱 |
| `auth.js` | 接口 `/api/auth`：注册 / 登录 | 前台登记处 |
| `projects.js` | 接口 `/api/projects`：**云端项目库**——每人独立的项目存/取/删 ★你说的「项目库」就是这个 | 私人保险柜 |
| `outlines.js`（api里的） | 接口 `/api/outlines`：大纲的读写（管理员可写、登录者可读），配合 `admin.html` | 目录模板的云端仓库 |
| `generate.js` | 接口 `/api/generate`：调 AI + 藏 API Key + 按用户每日限额 | 对外联络官 |
| `verify.js` | 旧版口令校验，**已不再使用**（可删可留） | 退役的旧门禁 |

> 后端还依赖两个**云端存储**（不在文件里，部署时在 Cloudflare 后台建）：**D1 数据库**（`users` 用户表 + `projects` 项目表，「项目库」的数据真正存这里）、**KV**（记每人每天用了几次）。建表脚本 `schema.sql` 这个压缩包里没带，需按 `部署说明.md` 手动执行。

**D. 数据 / 第三方库 / 文档（先不深究内部）**

| 文件 | 角色 | 类比 |
|---|---|---|
| `outlines.js`（根目录的） | 内置章节大纲数据（云端没配时的兜底） | 一本「目录模板册」 |
| `docx.umd.js` / `docxgen.js` | 导出 Word 用的第三方库 | 买来的零件 |
| `pdf.min.js` / `pdf.worker.min.js` / `mammoth.min.js` | 读/解析上传的 PDF、Word 用的第三方库 | 买来的零件 |
| `README.md` / `部署说明.md` | 怎么上线部署（含账号体系、D1、KV 配置） | 说明书 |

> 有两个都叫 `outlines.js`：根目录那个是**前端内置数据**（兜底），`functions/api/outlines.js` 是**后端接口**（云端大纲）。前台会优先用云端的，没有再退回内置的（见 `index.html` 的 `fetchOutlines`）。

**一句话架构**：
`index.html`（前端，纯 JavaScript 手写，无框架）在浏览器里跑，遇到「算钱」时调用三个 `*calc.js`，遇到「登录/存档/调 AI」时用 `fetch` 请求 `functions/api/` 里的后端函数，后端再去问数据库（Cloudflare D1）和 AI（DeepSeek）。

## Day 1 — 最基础语法（对照 Python）

**目标**：能读懂 JS 里的变量、函数、对象、数组、字符串拼接、`if/for`，知道它和 Python 的对应关系。**不求会写，只求会读。**

- 读：**讲义 01**。
- 对照代码：`rentcalc.js` 第 3 行（`r4` 函数）、第 14–37 行（`calc` 函数开头、`income` 计算）。这段几乎就是 Python 翻译成 JS，最适合入门。
- 核心问题（读完能答上来即过关）：
  1. JS 里 `const` / `let` 和 Python 的变量声明差别是什么？
  2. `x => x*2` 这种「箭头函数」等价于 Python 的什么？
  3. `{resi:0, park:0}` 是 JS 的什么？对应 Python 的什么？
  4. 为什么 JS 里到处是 `'...'+变量+'...'`？它在干 Python 的哪件事？
- 自测题（见讲义 01 末尾）。

## Day 2 — 整体设计 · 全流程大框架

**目标**：理解这个没有框架的网站是怎么「动」起来的——**状态变量 → 生成 HTML 字符串 → 塞进页面 → 重新绑定事件** 这个循环。这是全产品最重要的心智模型。

- 读：**讲义 02**。
- 对照代码：`index.html` 第 535–552 行（所有全局状态变量）、第 682–694 行（`renderSheet`，总调度）、第 660–678 行（`renderTOC`）、第 1638 行起（`bindEvents`）。
- 核心问题：
  1. `appMode`、`currentStep`、`scStep` 这几个全局变量分别控制什么？
  2. 用户点一下按钮，页面为什么会变？走了哪几步？
  3. 「首页 / 财务测算 / 可研生成」三条路是怎么分叉的？

## Day 3 — 核心思路①：财务计算引擎在想什么

**目标**：不看具体公式，先理解一个测算引擎的「骨架七步」：排年份 → 收入 → 成本 → 还本付息 → 税金 → 损益 → 现金流/IRR。

- 读：**讲义 03（上半：核心思路）**。
- 对照代码：`rentcalc.js` 的分节注释（第 22、39、66、96、114、122、142、157 行的 `===== N. xxx =====`）。**先只读注释，建立地图。**
- 核心问题：为什么现金流出「不含折旧和财务费用」？IRR 在解一个什么方程？

## Day 4 — 具体代码①：逐行读懂计算引擎

**目标**：把 Day 3 的骨架填上血肉，能逐行读懂收入、成本、还本付息迭代、IRR 牛顿迭代。

- 读：**讲义 03（下半：具体代码）**。
- 对照代码：`rentcalc.js` 全文；再对照 `salecalc.js`（你参与过的出售类）看「同一套骨架、不同公式」。
- 产出：在纸上或注释里，用中文把 `calc()` 的每一节写成一句话。

## Day 5 — 可研报告生成流程

**目标**：理解「六步流程」和 AI 逐章生成：怎么拼 prompt、怎么把真实测算数字喂给 AI、流式返回怎么解析、最后怎么导出 Word。

- 读：**讲义 04**。
- 对照代码：`index.html` 第 535 行（`STEPS`）、`stepDomain/stepProjectInfo/stepCalc/stepChapters/stepGenerate/stepReview`（函数表见讲义）、第 1447 行（`generateSection`）、第 1337 行（`buildCalcDigest`）、第 1612 行（`exportWord`）。
- 核心问题：AI 怎么保证「不编造财务数字」？「待填」模式和「真实数字」模式差在哪？

## Day 6 — 后端 · 账号 · 数据 · AI 代理

**目标**：理解为什么要有后端、Cloudflare 这套「Serverless」是什么、登录令牌怎么防伪造、密码怎么存、每日限额怎么算。

- 读：**讲义 05**。
- 对照代码：`functions/api/_auth.js`（全 56 行）、`auth.js`、`generate.js`、`projects.js`。
- 核心问题：
  1. API Key 为什么必须放后端、不能写在 `index.html`？
  2. 令牌（token）是怎么做到「服务器不存 session 也能验真伪」的？
  3. 密码为什么不存明文？PBKDF2 + salt 是防什么的？

## Day 7 — 整合复盘 + 练习

**目标**：把知识连成链路，并动手改一点点，检验是否真懂。

- 复盘练习（见讲义各自末尾汇总）：
  1. **串链路**：用户「注册 → 建项目 → 走测算 → 逐章生成 → 导出 Word」，每一步在代码里对应哪个函数、哪个后端接口？把这条链画出来。
  2. **改一个小东西**（任选其一，最能检验理解）：
     - 给出租类结果页新增一个指标 tile（照抄现有 `tile(...)` 写法）。
     - 把 AI 生成的字数要求从「500-800字」改成别的（找 `generateSection` 里的 prompt）。
     - 把每日限额 `USER_DAILY_LIMIT` 改一个数，想清楚它影响谁。
  3. **复述**：合上代码，用自己的话讲一遍「这个产品是怎么工作的」，能讲顺就算吃透。

## 学习小贴士

- **别怕 `index.html` 有 1900 行**：它 90% 是「函数们」，你一次只看一个函数，函数之间靠 Day 2 学的那个渲染循环串起来。
- **善用你的 Python**：JS 和 Python 是「近亲」，讲义 01 会把常见差异列成对照表，遇到看不懂的语法先回去查那张表。
- **看不懂就跳过细节，抓主干**：第一遍只求「知道这段在干嘛」，第二遍再抠语法。
- **想验证某段代码到底算出啥**：可以让我把某个函数单独跑一遍给你看输入输出，比干读快得多。
